# idsec_sca_discovery (Data Source)

Discover structural updates to an organization/directory that has already been onboarded to CyberArk, and scan for roles and resources

## Example Usage

```terraform
data "idsec_sca_discovery" "discovery_example" {
  csp             = var.csp
  organization_id = var.organization_id
  account_info = {
    id          = var.account_id
    new_account = var.new_account
  }
}
```

## Schema

### Required

- `account_info` (Attributes) A map of key-value pairs containing the account information of the workspace to discover (see [below for nested schema](#nestedatt--account_info))
- `csp` (String) The cloud provider that hosts the workspace to discover (AWS | AZURE | GCP)
- `organization_id` (String) The ID of the organization to discover (AWS - The AWS organization ID | AZURE: Microsoft Entra ID Directory (Tenant) ID | GCP: Google Cloud organization ID)
