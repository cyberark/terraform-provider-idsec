# idsec_sia_secrets_vm (Resource)

SIA Secrets VM resource, manages VM secrets information and metadata, based on the type of secret.

## Example Usage

```terraform
resource "idsec_sia_secrets_vm" "example_provisioner_secret" {
  secret_name          = "example_provisioner_secret"
  secret_type          = "ProvisionerUser"
  provisioner_username = var.provisioner_username
  provisioner_password = var.provisioner_password
}

resource "idsec_sia_secrets_vm" "example_pcloud_secret" {
  secret_name         = "example_pcloud_secret"
  secret_type         = "PCloudAccount"
  pcloud_account_name = var.account_name
  pcloud_account_safe = var.safe_name
}
```

## Schema

### Required

- `secret_name` (String) Name of the secret. For PCloudAccount type, this is auto-generated from account name and safe.
- `secret_type` (String) Type of the secret to add, data is picked according to the chosen type (ProvisionerUser,PCloudAccount)

### Optional

- `creation_time` (String) Creation time of the secret
- `is_active` (Boolean) Whether this secret is active or not and can be retrieved or modified
- `is_disabled` (Boolean) Whether the secret should be disabled or not
- `is_rotatable` (Boolean) Whether this secret can be rotated
- `last_modified` (String) Last time the secret was modified
- `pcloud_account_name` (String) If pcloud account type is picked, the account name
- `pcloud_account_safe` (String) If pcloud account type is picked, the account safe
- `provisioner_password` (String, Sensitive) If provisioner user type is picked, the password
- `provisioner_username` (String) If provisioner user type is picked, the username
- `secret` (Attributes) Secret itself (see [below for nested schema](#nestedatt--secret))
- `secret_details` (String) Optional extra metadata as JSON (use single quotes around JSON). Both secret types include 'certFileName', 'account_domain' (defaults to 'local'), 'ephemeral_domain_user_data'. Example: --secret-details '{"account_domain":"domain"}'. Additional fields are merged with defaults.
- `secret_id` (String) The secret id to change
- `tenant_id` (String) Tenant ID of the secret
